# Definition of Done

Задача считается завершённой, только когда выполнены **все** пункты. Проверка — обязательный чек-лист в шаблоне PR.

## Проектирование

- [ ] Для фичи есть связанный тикет / запись в бэклоге с привязкой к `FEATURE-XX` из ТЗ (`TZ_Telegram_Lead_Aggregator.md`).
- [ ] Для значимых архитектурных решений (новая внешняя интеграция, новый сервис, смена контракта БД/API) создан ADR в `docs/adr/` (см. ADR-0001).
- [ ] Definition of Ready соблюдён на момент начала работы (`docs/dor.md`).

## Реализация

- [ ] Реализация закрывает **все** критерии приёмки соответствующего FEATURE из ТЗ.
- [ ] Код проходит линтеры (`ruff`, `mypy` для Python; `eslint`, `tsc --noEmit` для frontend).
- [ ] Нет хардкода секретов, URL, API-ключей — только `.env` и `os.environ` / `process.env`.
- [ ] Логи не утекают PII (телефоны, session-файлы, тексты приватных сообщений вне `raw_messages`).

## Тесты

- [ ] Unit-тесты покрывают новый/изменённый код. Общее покрытие ветки ≥ **70%** (`pytest --cov`, `vitest --coverage`).
- [ ] Интеграционные тесты для каждой внешней границы, которую затрагивает фича: Telethon (через mock MTProto), Claude/OpenAI API (через recorded fixtures), Postgres, Redis, FastAPI endpoints.
- [ ] Миграции БД (`alembic`) протестированы: `upgrade head` → `downgrade -1` → `upgrade head` на копии prod-схемы без ошибок.

## Документация

- [ ] `README.md` затронутого модуля обновлён (если меняется публичный контракт или запуск).
- [ ] `CHANGELOG.md` содержит запись в формате Keep a Changelog.
- [ ] Если задача влияет на эксплуатацию — обновлён или создан runbook в `docs/runbook/`.
- [ ] Если меняется HTTP API — `docs/api/openapi.yaml` обновлён и прогнан через `openapi-spec-validator`.
- [ ] Если меняется LLM-промпт — новая версия в `prompts/vN/`, процесс — `docs/prompts-versioning.md`.

## CI/CD и ревью

- [ ] CI pipeline зелёный: lint, tests, build, openapi-validate, миграции.
- [ ] PR проревьюен минимум одним человеком, не автором.
- [ ] Конфликты с `main` разрешены через rebase.
- [ ] Деплой в dev прошёл, smoke-тест выполнен (или зафиксировано почему не требуется).

## Безопасность и операции

- [ ] Новые секреты добавлены в GitHub Environments (`dev`, `prod`) и задокументированы в `docs/security.md` (без значений).
- [ ] Изменения в зависимостях проверены `pip-audit` / `npm audit` без high/critical.
- [ ] Мониторинг/алерты добавлены для новой критичной логики (Sentry / метрики).
