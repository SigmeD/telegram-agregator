# Definition of Ready

Задача попадает в спринт, только если **все** критерии выполнены. Цель — исключить «разбираемся по ходу».

## Формализация требований

- [ ] У задачи есть название в императивной форме («Реализовать keyword-фильтр», не «Keyword-фильтр»).
- [ ] Описание содержит **контекст, цель, scope**. Scope зафиксирован явно: что **не** делаем в этой задаче.
- [ ] Привязка к `FEATURE-XX` из `TZ_Telegram_Lead_Aggregator.md`. Если задача не мапится ни на одну FEATURE — она либо инфраструктурная (помечается `chore`), либо не должна браться в спринт до обновления ТЗ.

## Критерии приёмки

- [ ] Явно перечислены acceptance criteria в формате чек-листа. Допустимые источники: соответствующий FEATURE из ТЗ, уточнения PM в тикете.
- [ ] Каждый критерий проверяем (measurable или binary). Формулировки вида «работает хорошо» — отклоняются.
- [ ] Нефункциональные ограничения из ТЗ разд. 4 (latency, uptime, стоимость) учтены, где применимо.

## Оценка и декомпозиция

- [ ] Задача оценена в днях, оценка ≤ **5 дней** на одного исполнителя. Задачи > 5 дней обязательно декомпозируются (split).
- [ ] Если задача затрагивает несколько контекстов (backend + frontend + infra) — декомпозирована на подзадачи по контекстам с отдельными PR.

## Зависимости

- [ ] Явно перечислены зависимости: другие задачи, внешние API-ключи, доступы, данные, решения от PM/заказчика.
- [ ] Все блокирующие зависимости разрешены на момент планирования. Не-блокирующие — помечены.
- [ ] Если требуется новое ADR — он должен быть создан **до** начала реализации (статус Proposed достаточно для старта).

## Дизайн и данные

- [ ] Если меняется схема БД — есть эскиз DDL в описании задачи или в связанном ADR.
- [ ] Если меняется HTTP API — эскиз endpoint'а (метод, path, request/response) в описании.
- [ ] Если требуются тестовые данные — указан источник (fixtures, seed-скрипт, ручная разметка PM).

## Команда

- [ ] Исполнитель назначен.
- [ ] Ревьюер известен (не обязательно назначен в GitHub до PR, но договорённость есть).
- [ ] PM подтвердил приоритет задачи в текущем спринте.
